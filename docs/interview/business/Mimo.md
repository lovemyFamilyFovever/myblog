## 一、先说你提到的那些软件

你之前以为它们都要花钱，其实大部分都有免费方案，而且它们的性质不太一样：

| 名称 | 本质 | 费用 |
|---|---|---|
| **Claude Code** | Anthropic 官方的终端编码代理，能读写你本地文件 | 需要 Anthropic API key，按用量付费，不便宜 |
| **Codex** | OpenAI 新出的云端编码代理 | ChatGPT Pro 用户可用 |
| **OpenCode** | 开源的终端编码工具，类似 Claude Code 的平替 | 免费，但需要你自己接 API key |
| **KiloCode** | VS Code 插件，开源的 AI 编码助手 | 免费，接自己的 API key |
| **Hermes Agent** | 基于开源模型的智能体 | 免费 |
| **OpenClaw** | 应该是 MiMo Claw 相关的 | 你作为米粉应该比我更清楚 |

**关键认知**：这些不是"大模型"本身，而是**调用大模型来帮你写代码的工具**（业内叫 AI Coding Agent）。大模型是底层的引擎（比如 Claude、GPT、MiMo），这些工具是装了引擎的车。你之前用的 VS Code + Copilot/Fitten Code/通义灵码，其实也是同类的东西，只是它们做得更轻量。

你已经用过通义灵码的独立客户端了，那个体验就是"AI 编码代理"的基本形态——能直接改你的文件。Claude Code、OpenCode 这些做的事情类似，只不过它们跑在终端里，面向的是更习惯命令行的开发者。

**你现在不需要全部搞懂，先把手头用得顺的工具用熟就够了。** MiMo 既然你觉得编程能力强，就继续用它。等你真正需要更复杂的编码代理能力时，再按需学习。
