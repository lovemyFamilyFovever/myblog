

## 中文思考过程

**不管它底层是怎么计算的，只要它展示出来的“思考过程”文本，必须是中文的。**

### 选项一：最精准、最柔和（推荐作为系统提示词或放在问题最前面）
这个版本明确告诉模型：我不干涉你内部怎么想，但我要求你“呈现”出来的文字必须是中文。

```txt
请注意规范你的输出格式：当你展示‘思考过程’时，该区域的文本必须全部使用简体中文呈现。
你可以使用任何语言逻辑进行推演，但在将思考步骤输出到屏幕上时，
请确保将其转化为中文表达。代码和专有名词可保留英文，但解释和推导过程必须是中文。
```

### 选项二：针对特定标签（如果你用的模型有 `<thinking>` 这样的折叠标签）
很多带思考过程的模型（如 DeepSeek-R1、某些 Claude 套壳前端）会使用特定的标签来包裹思考过程，直接对标签内部做要求效果最好。

```txt
在输出你的思考过程（如 `<thinking>` 标签内的内容）时，请务必使用纯简体中文进行书写。
请不要在该展示区域内输出英文句子，只需将你的推理步骤用中文翻译后展示出来即可。
```

### 选项三：极简版（适合每次提问时顺手加在末尾）
如果你不想写太长，只需一句话强调“显示层面”的要求：

```txt
补充要求：请确保你输出的【思考过程/草稿】部分以纯中文显示。
```

## Bug修复

把报错信息和相关代码喂给它，并加上你定制的规则。

> **提示词：**
> “我遇到了一个 Bug，请帮我排查。
> 
> **技术栈：** `[Vue3 + Composition API + Pinia]
> 
> **报错信息：** `[粘贴控制台完整的错误栈]`
> 
> **相关代码：** `[粘贴你认为出问题的组件代码和 Store 代码]`
> 
> **系统指令**：
```txt
请不要直接给我最终修复代码。先分析导致该 Bug 的可能原因，然后给我 2-3 种不同的解决选项，
针对每个选项说明优缺点和潜在影响。最后等我选择后，你再输出具体代码。”
```

## 翻译设计稿/图片（模拟前端视角）
大模型的多模态能力很适合用来解析 UI。

> **提示词：**
> “我将上传一张 `[网页截图 / UI 设计稿]`。
> 请作为一名高级前端工程师分析这张图：
> 1. 拆解它的页面结构（Header, Main, Sidebar 等），用嵌套列表的形式写出 DOM 结构树。
> 
> 2. 分析它的配色方案（提取主色、辅色、背景色、文字颜色的 Hex 值）。
> 
> 3. 指出里面有哪些交互元素（按钮、输入框、弹窗），并推测它们的交互状态。
> 
```txt
给出示例代码，并给出注释，说明你的代码逻辑和思路。
```



```txt
执行操作前如果不清楚请跟我沟通确认。
```
